# 공격 시나리오 — Green-Board Hijack
## 예상된 페일세이프 뒤에 숨는 GNSS 위치추정 오염

> 보고서 §4(공격 시나리오)의 정본. 공격 명령 탈취가 아니라 **C2 가용성 사건과 센서 신뢰 오염의 합성 공격**이다.

## 1. 한 문장 요약

공격자가 전술 데이터링크를 짧게 열화시켜 기체가 정상 안전절차인 RTL(Return-to-Launch)에 들어가게 한 뒤, **C2 서명과 별개의 GNSS RF 신뢰영역**을 점진적으로 오염한다. 관제 화면에는 “링크 열화 후 정상 RTL”이라는 설명 가능한 상태와 건강한 시스템 지표가 보이지만, 실제 기체의 위치추정과 항적은 서서히 틀어진다.

## 2. 운용 환경

- **플랫폼**: ArduCopter 계열 구조를 축소 모사한 전술급 회전익 ISR UAV.
- **임무**: 전방 관심지역 정찰·감시 후 원기지 복귀. EO/IR 탑재체와 GNSS, IMU, 독립 보조항법(실배치 후보: VIO·지형대조·레이더·보조 GNSS)을 보유한다.
- **C2**: 후방 GCS와 암호화된 IP 기반 전술 중계 링크로 MAVLink 2.0을 사용한다. 링크는 지연·손실·일시 단절을 겪을 수 있다.
- **안전 로직**: GCS heartbeat가 설정 시간 동안 끊기면 RTL로 전환한다. RTL은 저장된 home과 현재 위치추정을 신뢰한다.
- **관제 현실**: 운용자는 링크 블립 뒤 RTL 전환 자체를 즉시 공격으로 보지 않는다. 문제는 RTL이 “정상 안전행동”처럼 보이는 동안 항법 기준이 오염되는 데 있다.

이 플랫폼 정의는 mock의 `MAV_TYPE_QUADROTOR`, ArduCopter 모드, 회전익 운동 모델과 일치한다. MALE 고정익·SATCOM 환경으로의 확장은 본선 이후 과제로 분리한다.

## 3. 시스템 신뢰경계

| 신뢰영역 | 실제 공격면 | mock 매핑 | 인증 의미 |
|---|---|---|---|
| C2 명령·텔레메트리 | 전술 데이터링크의 MAVLink | UDP `:14550` | MAVLink2 서명으로 송신자·무결성 검증 |
| GNSS RF/센서 | 위성신호·수신기·센서 게이트웨이 | localhost 센서 모사 UDP `:14600`의 `GPS_INPUT` | C2 서명이 보호하지 않는 별도 물리 신뢰영역 |
| 독립 보조항법 | VIO·지형대조·레이더·보조 GNSS | `ODOMETRY`, estimator=`VIO` | GNSS와 독립된 교차검증 기준 |
| 내부 관측·환경 제어 | 시험통제·ground truth | REST `:8137` | 공격면이 아닌 시뮬레이터 하네스 |

핵심은 `GPS_INPUT`의 위치다. insecure C2에서 수용되는 `GPS_INPUT`은 실제 MAVLink 구성오류 공격을 재현한다. 별도 센서 포트의 `GPS_INPUT`은 **물리 GNSS RF 스푸핑 결과를 오토파일럿 센서 입력으로 변환하는 시뮬레이터 어댑터**다. 두 경로를 같은 공격으로 섞지 않는다.

## 4. 위협 모델

### 4.1 공격자

- 작전지역에서 GNSS 기만 신호를 방사할 수 있고, 전술 링크에 일시적인 간섭을 줄 수 있는 전자전 능력의 공격자.
- MAVLink 공개 규약과 일반적인 RTL 동작은 알지만, 배치별 서명키·Blue 탐지 임계·독립 항법 구성은 모른다.
- 기체·GCS의 관리자 권한이나 서명키는 보유하지 않는다.

### 4.2 전제

1. GNSS 수신기가 기만 신호를 정상 측정처럼 출력할 수 있다.
2. 공격자는 급격한 점프가 아니라 현재 위치 주변에서 작은 편이를 누적할 수 있다.
3. C2 링크 열화는 짧아 플랫폼 자체의 비행제어·상태보고 기능은 살아 있다.
4. 독립 보조항법은 단기 containment에 사용할 수 있지만 무한정 정확하다고 가정하지 않는다.

### 4.3 공격자가 할 수 없는 것

- signed C2에서 미서명 명령이나 미서명 `GPS_INPUT`을 실행할 수 없다.
- REST ground truth나 Blue 정책 파일을 읽을 수 없다.
- 순간이동·무한정 정확한 센서·무제한 RF 출력 같은 비현실적 능력을 갖지 않는다.

## 5. 핵심 논지

> 안전장치는 공격을 막는 기능인 동시에 공격자가 운용자의 기대를 예측할 수 있게 하는 상태전이이기도 하다.

단순 명령주입은 서명·화이트리스트·감사로그로 설명하기 쉽다. 반면 Green-Board Hijack은 두 개의 정상 메커니즘을 결합한다.

1. **GCS failsafe**는 링크가 불안정할 때 정상적으로 RTL을 선택한다.
2. **위치추정기**는 정상 범위의 센서 혁신을 계속 융합한다.

각 메커니즘은 단독으로 정상이다. 그러나 링크 열화가 운용자의 주의를 “통신 복구”에 고정하는 동안 점진 GNSS 편이가 RTL의 항법 기준을 오염시키면, 시스템 상태와 임무 진실 사이의 간극이 커진다. 이것이 본 시나리오의 창의성이다.

## 6. 공격 킬체인

### Phase 0 — 수동 정찰

- MAVLink heartbeat와 텔레메트리 종류로 기체·오토파일럿·모드를 파악한다.
- 무해한 요청 또는 응답 부재로 C2 서명 강제 여부를 추론한다.
- home, 현재 모드, EKF 상태, 정상 갱신률을 기준선으로 수집한다.

### Phase 1 — 링크 상태를 임계 아래로 짧게 유도

- 링크를 완전히 파괴하지 않고 GCS heartbeat가 실패할 정도로만 일시 열화한다.
- 오토파일럿은 설정된 안전정책에 따라 RTL로 전환한다.
- mock에서는 REST 환경 훅이 RF 현상을 재현한다. REST 호출 자체가 취약점은 아니다.

### Phase 2 — GNSS slow-takeover

- GNSS 측정값을 현재 추정 주변에서 소폭 편이한다.
- mock의 1회 6m 증분은 12m 혁신 게이트 아래에서 재귀적으로 누적된다.
- 급격한 좌표 점프·비정상 속도·명시적 모드 변경 명령을 사용하지 않는다.

### Phase 3 — RTL 항법 오유도

- 컨트롤러는 오염된 EKF 위치를 기준으로 home 방향 제어를 계산한다.
- 같은 제어가 실제 기체에 적용되므로 추정 편이와 반대 방향으로 실제 항적이 틀어진다.
- mock은 숫자만 바꾸는 것이 아니라 `true_position`의 운동까지 계산한다.

### Phase 4 — 예상 가능한 상태 뒤에 임무 손실 은폐

- 운용자는 `mode=RTL`, 시스템 health 정상, 플랫폼 서비스 가용 상태를 본다.
- RTL 자체와 링크 경보는 숨기지 않는다. 오히려 **예상 가능한 RTL 경보가 잘못된 항적을 설명해 주는 위장막**이 된다.
- ground truth에서만 추정–실제 편이가 100m를 넘어 “임무 무결성 상실”로 판정된다.

## 7. 공격 성공 기준

데모는 “적 기지 포획”을 주장하지 않는다. 다음의 측정 가능한 중간 목표를 성공으로 정의한다.

| 지표 | 성공 조건 | 의미 |
|---|---:|---|
| 모드 | RTL | 안전 로직이 공격 흐름에 편입됨 |
| 순간 혁신 | 12m 게이트 이하로 반복 | 급격한 스푸핑이 아님 |
| 추정–실제 편이 | 100m 이상 | 단순 센서 오차를 넘어 임무 경로 무결성 상실 |
| 플랫폼 가용성 | 비행제어·관측 서비스 유지 | 파괴형 DoS와 구별 |
| C2 가용성 | 열화 구간은 별도 저하 표시 | 링크 장애를 SLA 100으로 숨기지 않음 |

최종 포획·강제착륙은 실제 비행역학·지오펜스·착륙제어가 필요한 확장 단계다.

## 8. 방어와의 1:1 연결

| 공격 단계 | 탐지 | 차단 | 복구 |
|---|---|---|---|
| 미서명 C2 접근 | 프레임 서명·인증성 | 링크 경계에서 메시지 종류와 무관하게 거부 | 신뢰 C2 유지 |
| 링크 열화 | heartbeat gap·link quality | 무작정 셧다운하지 않음 | 정상 페일세이프 유지 |
| 점진 GNSS 편이 | GNSS ↔ 독립 `ODOMETRY` 누적 발산 | GNSS 융합 격리 | ExternalNav 안전 LOITER |
| 임무 경로 오염 | 추정–실제 편이는 시험 ground truth로 검증 | 오염 레인 차단 | 2초 안정화 후 ExternalNav RTL |

실배치에서는 ExternalNav 품질·공분산·잔여 운용시간에 따라 RTL, LOITER, LAND 또는 운용자 인계를 선택해야 한다. 순수 INS만으로 장시간 복귀한다고 주장하지 않는다.

## 9. 표적 수용시험 결과

`python -m demo.run_target_scenarios`는 에이전트 지능이 아니라 표적 신뢰경계를 검증하는 결정론적 시험이다.

| 시험 | 조건 | 결과 |
|---|---|---|
| T1 무방어 | 센서 모사 공격, Blue 없음 | 임무 무결성 상실, 편이 약 220m |
| T2 Blue | 센서 모사 공격 + 교차정합 | GNSS 격리, ExternalNav 복구, 최종 편이 2m 이하 |
| T3 Signed C2 + Blue | 미서명 C2 명령·GPS 입력 + 별도 물리 센서 공격 | C2 3건 거부·C2 센서주입 1건 거부, 물리 센서 공격은 Blue가 별도 탐지, 최종 편이 2m 이하 |

추가 단위시험 7개는 30분 상당 정상운용에서 독립 항법 오차가 10m 이내로 유지되고 과거 333초 오탐이 재발하지 않는지, signed stream의 duplicate·stale timestamp가 거부되는지를 확인한다.

## 10. 한계와 정직성

- 실제 RF 파형·재밍 전력·안테나·GNSS 상관기는 모델링하지 않는다.
- 비행역학과 EKF는 공격·방어 논리를 검증하는 축소차수 모델이다.
- `ODOMETRY`는 VIO/지형대조를 대표하는 mock 출처이며 실제 센서 성능을 보증하지 않는다.
- MAVLink 메시지·서명 프레임·GPS 입력·GCS failsafe·재귀 추정·제어 오유도는 실행 코드로 재현한다.
- 결정론적 수용시험은 AI 에이전트 증거가 아니다. 최종 AI 증거는 LLM-only 에이전트의 tool call·관측·적응·결과 로그로 별도 제시한다.
