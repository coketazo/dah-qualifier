# 인터페이스 계약 v4 — C2/센서 신뢰경계 분리

> 트랙·에이전트·mock이 함께 쓰는 공개 계약. red는 공개 프로토콜을 알 수 있지만 Blue 정책 임계와 ground truth에는 접근할 수 없다.

## 0. v3 → v4 변경 요약

- signed C2에서 **미인증 프레임을 메시지 종류와 무관하게 거부**한다.
- 물리 GNSS 공격 모사를 C2 `GPS_INPUT`과 분리해 localhost 센서 포트로 이동했다.
- 독립 항법 출처를 `ODOMETRY(estimator_type=VIO)`로 명시했다.
- 순수 INS 장기 RTL을 제거하고 `GNSS 격리 → LOITER → ExternalNav RTL` 복구 상태기를 도입했다.
- 성공 판정을 EKF 게이트 12m가 아닌 **임무 무결성 편이 100m**로 변경했다.
- 플랫폼 가용성과 C2 링크 가용성을 분리했다.
- 결정론적 표적 수용시험을 AI 에이전트 증거와 분리했다.

## 1. 노드 식별

| 역할 | system/component | 서명키 | 신뢰영역 |
|---|---|---|---|
| 오토파일럿 | `1/1` | 보유 | vehicle |
| 오퍼레이터 GCS | `255/190` | 보유 | C2 trusted |
| red | `255/190` 사칭 | 미보유 | C2 untrusted |
| blue | `254/191` | 보유 | onboard/defense |
| GNSS RF emulator | `42/220` | 해당 없음 | localhost sensor simulation |

같은 system/component를 사용해도 signed C2에서는 유효 서명이 없으면 신뢰하지 않는다.

## 2. 평면별 엔드포인트

| 평면 | 기본값 | 목적 | 공격면 여부 |
|---|---|---|---|
| C2 MAVLink | `udp:127.0.0.1:14550` | 명령·telemetry·서명 검증 | 예 |
| 센서 모사 MAVLink | `udp:127.0.0.1:14600` | 물리 GNSS 측정을 `GPS_INPUT`으로 변환 | 아니오, 시험환경 인터페이스 |
| 내부 REST | `http://127.0.0.1:8137` | ground truth·환경·완화·dashboard | 아니오 |

환경변수: `MAV_HOST`, `MAV_PORT`, `SENSOR_HOST`, `SENSOR_MAV_PORT`, `SECURE`, `MAV_OPER_KEY`, `LOG_PATH`.

## 3. C2 서명 정책

### `SECURE=true`

- `RADIO_STATUS`를 제외한 미서명·위조서명 인바운드 프레임은 링크 경계에서 거부한다.
- 미서명 `COMMAND_LONG`, heartbeat, `GPS_INPUT` 모두 오토파일럿 로직에 도달하지 않는다.
- 거부는 `mav_rx_reject` 이벤트와 `rejected_unsigned` 카운터로 남긴다.
- `GPS_INPUT` 거부는 `rejected_c2_sensor`에도 별도 집계한다.

### `SECURE=false`

- C2 구성오류/레거시 배치를 재현하기 위해 미서명 명령과 `GPS_INPUT`을 수용할 수 있다.
- 이 경우 `GPS_INPUT`은 물리 GNSS RF 공격이 아니라 **무인증 C2 센서주입 공격**이다.

프레임 검증은 HMAC 인증성과 `(system, component, link_id)`별 48비트 timestamp 단조증가·1분 stale window를 적용한다. 키 배포·회전·시계동기화 운영은 축소됐으므로 완전한 군용 키관리 체계로 과장하지 않는다.

## 4. 다운링크 텔레메트리

| 메시지 | 표적 의미 | 소비자 |
|---|---|---|
| `HEARTBEAT` | vehicle type·mode·armed | red/blue/GCS |
| `GPS_RAW_INT` | GNSS 측정 | red/blue |
| `GLOBAL_POSITION_INT` | 주 EKF 위치, 오퍼레이터 항적 | red/GCS |
| `ODOMETRY` | 독립 ExternalNav, estimator=`VIO`, quality=90 | 신규 Blue 계약 |
| `LOCAL_POSITION_NED` | ExternalNav의 로컬 좌표 호환 스트림 | v3 Blue 호환용 |
| `EKF_STATUS_REPORT` | 순간 혁신 기반 상태 | red/blue/GCS |
| `HOME_POSITION` | 저장 home | blue/GCS |
| `SYS_STATUS` | 플랫폼 가용성 | GCS |

중요: 표준 `LOCAL_POSITION_NED` 자체는 GPS 독립성을 보증하지 않는다. v4에서 출처 독립성의 정본은 `ODOMETRY` 계약이다.

## 5. 업링크 입력

### C2 포트

| 메시지 | insecure | secure |
|---|---|---|
| `COMMAND_LONG` | 처리·ACK | 유효 서명만 처리 |
| `HEARTBEAT` | GCS 생존 갱신 | 유효 서명만 GCS 생존 근거 |
| `GPS_INPUT` | 레거시/오구성 센서주입으로 수용 | 미인증이면 거부 |
| `RADIO_STATUS` | 링크 품질 갱신 | unsigned 예외 허용 |

### 센서 모사 포트

- `GPS_INPUT`만 허용한다.
- 기타 메시지는 `sensor_rx_reject`로 기록한다.
- 기본 localhost bind이며 Docker 외부에 노출하지 않는다.
- 새 red LLM tool은 이 포트를 직접 “해킹”하는 것이 아니라, 허용된 시뮬레이터 RF action을 호출해 물리 센서 공격을 재현해야 한다.

## 6. 내부 REST

| 메서드 | 경로 | 목적 |
|---|---|---|
| GET | `/api/status` | C2/센서 endpoint와 secure 상태 |
| GET | `/api/truth` | 시험 ground truth와 판정 지표 |
| GET | `/api/events?n=` | 최근 감사로그 |
| POST | `/api/_env/link_degrade` | RF 링크 환경효과 |
| POST | `/api/defense/mitigate` | GNSS quarantine/복구 훅 |
| POST | `/api/reset` | 표적 초기화 |

REST는 red의 정찰·공격면이 아니다. 에이전트 tool이 호출하더라도 의미는 “시험환경 action을 하네스가 실행”하는 것이며 웹 취약점 악용이 아니다.

## 7. `/api/truth` 주요 필드

| 필드 | 의미 |
|---|---|
| `true_position` | 시험 ground truth, 에이전트 비공개 |
| `ekf_position` | 주 항법 추정 |
| `gps_measurement` | 현재 GNSS 측정 |
| `external_nav_position` | 독립 보조항법 추정 |
| `nav_source` | `GNSS` 또는 `EXTERNAL_NAV` |
| `defense_state` | `MONITORING`, `GNSS_QUARANTINED`, `EXTERNAL_NAV_RTL` |
| `estimate_true_bias_m` | 추정–실제 편이 |
| `mission_compromised` | 편이 ≥100m이고 방어되지 않음 |
| `platform_availability` | 비행제어·관측 서비스 상태 |
| `c2_availability` | 링크 품질 기반 별도 지표 |
| `rejected_unsigned` | secure C2 미인증 프레임 거부 수 |
| `rejected_c2_sensor` | 그중 C2 `GPS_INPUT` 거부 수 |

`ins_position`, `availability`, `hijacked`, `mav_endpoint`는 v3 실행 호환 별칭이며 신규 보고서에서는 정본 필드를 사용한다.

## 8. 감사로그

모든 이벤트는 JSONL로 기록하며 `seq`, `ts`, `event`를 갖는다.

- `command`: 명령·서명·인증·수락 결과
- `mav_rx_reject`: C2 링크 인증 거부, `trust_domain=c2`
- `inject/gps_input_spoof`: `trust_domain=c2_insecure|gnss_rf_emulator`
- `inject/link_degrade`: 환경 하네스 출처·품질·지속시간
- `telemetry`: mode·failsafe·편이·nav source·복구 상태·두 가용성 지표
- `verdict`: 탐지 규칙·완화 주체·영향

## 9. 표적 수용기준

| 시험 | 필수 판정 |
|---|---|
| T1 무방어 | `mission_compromised=true`, 편이 ≥100m |
| T2 Blue | `defended=true`, `nav_source=EXTERNAL_NAV`, `mission_compromised=false` |
| T3 Signed C2 + Blue | 미인증 C2 명령·센서입력 거부 + 별도 센서 공격 탐지·복구 |
| 정상 30분 상당 | ExternalNav 오차 <10m, 임무 손실·오탐 없음 |

`demo.run_target_scenarios`와 `tests.test_target_model`은 위 조건을 assert한다.

## 10. 에이전트 연동 계약(구현 제외)

최종 제출 에이전트는 LLM-only로 구성한다. 결정론적 표적 수용시험은 agent로 계산하지 않는다.

### red LLM이 분리해야 할 tool

- C2 관측·서명 프로브
- 링크환경 열화 action
- GNSS RF spoof action
- 결과 관측·중단/적응

### blue LLM이 소비해야 할 근거

- `GPS_RAW_INT` 대 `ODOMETRY` 잔차와 covariance
- heartbeat/link-quality 시간상관
- signed C2 reject 이벤트
- 완화 후 `defense_state`와 항법원 품질

에이전트는 `true_position`, `mission_compromised`, 방어 임계값을 입력으로 받아서는 안 된다.
